\section{\RG, a Discrete-Logarithm Based DKG} \label{storm-gen}
Finally, we instantiate our generic DKG presented in Section~\ref{generic-construction} with a concrete construction, \RG.
Due to the proof of security for the generic construction,
\RG is secure under the Computational Diffie-Hellman (CDH) problem in the random oracle model,
 in the honest majority setting.

\paragraph{Target Key Generation}
The target key generation scheme $\targetkeygen$ for \RG is defined as follows.
The secret domain $\secretspace$ is the field $\Zq$ and the public domain $\publicspace$ is the group $\GG$ generated by $g$.
\begin{itemize}
  \item $\keygen(\lambda) \rightarrow (\SK, \PK)$: Sample $\SK \rgets \Zq$. Generate $\PK \gets g^{\SK}$. Output $(\SK, \PK)$.
  %\ian{$\lambda$ is unused here, and relatedly, $\GG$ and $q$ are undefined.}
    %\chelsea{$\GG$ and $\q$ are defined in the preliminaries.}
    %\chelsea{The precise way to handle $\lambda$ is to have a parameter
    %generation algorithm, but I'm not going to do that now.}
  \item $\verify(\SK, \PK) \rightarrow \zo$: Output $1$ if $g^{\SK} = \PK$. Otherwise, output $0$.
\end{itemize}

\paragraph{Building Blocks.}
We next describe the concrete building blocks required for \RG:

\begin{enumerate}
  \item the \ENIKE scheme presented in Section~\ref{enike:concrete},
  \item the DL-based \AVSS scheme presented in Section~\ref{avss:concrete}, and
  \item a secure hash function $\Hash$, where
$\Hash_1$, and $\Hash_2$ are domain-separated instances of $\Hash$.
 These functions are defined as follows:
\begin{itemize}
  \item $\Hash_1:  (\GG^t)^n  \times \Zq \rightarrow \Zq$: Accepts as input a vector, where the first element is a tuple
  of $n$ \AVSS commitments $\vsscommit_j \in \GG^t, j \in \set{n}$ which are each a tuple of group elements of size $t$,
  and the second element is an auxiliary element $\auxstring \in \Zq$.
\item $\Hash_2:  (\GG)^n \rightarrow \Zq$: Accepts as input a vector
  of $n$ blinding factors $\blinding_j \in \GG, j \in \set{n}$,
  and outputs an auxiliary element $\auxstring \in \Zq$.
\end{itemize}
\end{enumerate}

\paragraph{Correctness and Security.}
\RG is an instantiation of the generic construction presented in Section~\ref{generic-construction},
such that Assumption~\ref{ass:avss-enike} holds for the DL-based \AVSS scheme presented in Section~\ref{enike:concrete}
and the DL-based \AVSS scheme presented in Section~\ref{avss:concrete}.
For this reason,
the correctness and security of \RG follows from the proofs for the generic scheme.

For security,
by Theorem~\ref{thm:perfect-robust}
\RG is weakly robust,
assuming the \AVSS is zero-indexed and secure,
and the NIKE is binding.
We establish the security of the zero-indexed DL-based \AVSS scheme in Section~\ref{appendix:avss-proofs},
and Theorem~\ref{thm:enike-binding} establishes that the concrete NIKE of Section~\ref{enike:concrete} is binding.

By Theorem~\ref{thm:zero-knowledge},
\RG is zero-knowledge,
assuming the \AVSS achieves aggregated secrecy and the NIKE is unrecoverable.
We establish that the zero-indexed DL-based \AVSS scheme indeed achieves aggregated secrecy in Section~\ref{appendix:avss-proofs}.
The unrecoverability of the concrete NIKE of Section~\ref{enike:concrete} is a direct reduction to CDH.
