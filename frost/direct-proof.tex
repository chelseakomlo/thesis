\section{Security of FROST}
\label{frost:direct-proof}

We now present a direct proof of static (weak) unforgeability for the FROST signature construction given in Figure~\ref{fig:frost}.
This proof builds on prior proofs of security for FROST~\cite{BellareCKMTZ22},
but gives a reduction to \AOMDL, as opposed to \OMDL.
As discussed in Section~\ref{assumptions},
\AOMDL is a falsifiable and hence weaker assumption than \OMDL.
Note that prior work also established strong unforgeabiity of FROST~\cite{BellareCKMTZ22},
but with respect to \OMDL.

Our proof assumes a centralized key generation mechanism.



\begin{theorem}\label{thm:FROST-aomdl}
	FROST is statically unforgeable   against an adversary $\adv$ that is allowed to corrupt up to $\tcor$ participants,
  assuming the \AOMDL assumption holds for $\GroupGen$ in the ROM.
  In other words, for every static unforgeability adversary $\adv$ that wins the (static) unforgeability game as given in Definition~\ref{defn:thresh-static-unforg},
  there exists a PPT adversary $\ddv$ that wins the \AOMDL game.

  The advantage of $\adv$ is bounded by


  \begin{equation} \label{eq:frost-aomdl}
\advant{$\uf$}{\frost,\A}(\secp, n, \thresh, \tcor)
    \leq \sqrt{\rototal \cdot \advant{aomdl}{\ddv}(\secp) } + \mathsf{negl}(\secp)
  \end{equation}

\noindent such that $\rototal = \qnon + \qsigh +  \qsig + 1$ is the number of times $\adv$ is allowed to query the random oracle,
where $\qnon$ is the number of times $\adv$ is allowed to query $\HashNon$,
$\qsigh$ is the number of times $\adv$ is allowed to query $\HashSig$,
and $\qsig$ is the number of times $\adv$ is allowed to query its signing oracle.

\end{theorem}

To prove Theorem~\ref{thm:FROST-aomdl},
we rely on two lemmas.

\begin{lemma} \label{lem:frost-simulation}
There exists an algorithm $\SIM$ which can simulate FROST with respect to an \AOMDL instance to an adversary $\adv$
playing against the unforgeability game as in Figure~\ref{fig:euf-cma-threshold}.
In other words,
$\adv$ has negligible additional advantage in winning the unforgeability game when
interacting with $\SIM$ that it does when playing against FROST directly.

Concretely, in the setting where $\adv$ is allowed up to $\tcor$ corruptions, then
\begin{align} \label{eq:simulation-frost}
  \begin{split}
\advant{$\uf$}{\frost,\A}(\secp, n, \thresh, \tcor)  & \leq
        \Pr [\ \SIM^\adv(\pp, n, t, t-1, \Inst) \neq \bot \ ]
        + \frac{\rototal}{q}
        + \frac{\rototal^2}{2q}
  \end{split}
\end{align}
\end{lemma}

\begin{lemma} \label{lem:frost-reduction}
Given any PPT adversary $\adv$ that is allowed to corrupt $\tcor$ participants and wins the unforgeability game against FROST,
and a simulating algorithm $\SIM$ which can simulate FROST,
then there exists a PPT \AOMDL adversary $\ddv$ that can use $\adv$ as a subroutine to solve for its \AOMDL challenge.

Concretely,
    \begin{align} \label{eq:ddv-dl}
    \begin{split}
      \advant{aomdl}{\ddv^{\SIM}}(\pp, n, t, t-1, \Inst) & \geq
        \frac{\advant{uf}{\frost,\adv}(\lambda, n, t, t-1)^2}{\rototal}
        - \frac{2\rototal}{q}
        - \frac{3\rototal^2}{q}
        - \frac{2\qsig}{q}
    \end{split}
    \end{align}
\end{lemma}

\begin{proof}[Proof of Lemma~\ref{lem:frost-simulation}]
  We give the proof by one game hop and then describing the simulator $\SIM$.

  \paragraph{Game~0.}
  This is the unforgeability game as defined in Definition~\ref{defn:thresh-static-unforg},
  instantiated with FROST.

  Let $\A$ be an adversary playing the static unforgeability game.
  $\A$ is allowed to query $\HashNon$ $\qnon$ number of times,
  $\HashSig$ $\qsigh$ number of times,
  and is allowed to make $\qsig$ signing queries.
  Without loss of generality, we assume $\A$ queries $\HashSig$ on its forgery $(\aggR^*, \threshpk, m^*)$.
  Finally, let $\rototal = \qnon + \qsigh + \qsig + 1$.

  Let $W_0$ be the event that $\adv$ wins Game~0.
  Then, the advantage of $\adv$ is simply

  \begin{equation} \label{eq:game0-frost}
    \advant{uf}{\frost, \adv}(\lambda) = \Pr[W_0]
  \end{equation}

  \paragraph{Game~1.}
  The only difference between Game~0 and Game~1 is as follows.
  If $\adv$ queries $\HashSig$ or $\HashNon$ on any input that produces a colliding output with another query,
  then the environment aborts.

  \paragraph{Difference between Game~1 and Game~0.}
  Let $W_1$ be the event that $\adv$ wins Game~1.
  Because we model $\HashSig$ and $\HashNon$ as random oracles,
  then

  \begin{equation} \label{eq:game1-frost}
    \lvert \Pr[W_1] - \Pr[W_0] \rvert \leq \frac{\rototal^2}{2q}
  \end{equation}

  \paragraph{Game~2.}
  We now describe a simulating algorithm $\SIM$ that simulates FROST to  $\adv$.

  \medskip

  \noindent\textit{Setup.}
  $\SIM$ is initialized with public parameters $\pp = (\Gr, q, g)$,
  threshold parameters the total number of parties $n$,
  threshold $t$,
  and corruption threshold $t-1$.
  Additionally,
  $\SIM$ is initialized with the instance $\Inst$,
  which is comprised of
  a set  of $2\qsig+1$ \AOMDL challenges $\{ X_0, \ldots, X_{2\qsig}\}$,
  and a set of values
  $\{h_1, \ldots, h_{\rototal} \}$ to program its random oracles.
  $\SIM$ can query the \AOMDL solution oracle $\odlsol$ up to $\qsig$ times.

  \medskip

  $\SIM$ then initializes the following values:
  \begin{enumerate}
    \item tables $\QNon \gets \emptyset$, $\QSig \gets \emptyset$ to simulate random oracles $\HashNon, \HashSig$;
    \item table $Q_1 \gets \emptyset$ to maintain state during its simulation;
    \item tables $W, W'$ to maintain open and closed sessions,
    \item the table $Q_m \gets \emptyset$ to maintain the set of messages
queried by $\A$ to honest signers,
    \item the counter $\counter \gets 1$ to keep track of which \AOMDL challenges have yet been used.
  \end{enumerate}

  $\SIM$ runs $\A(\pp, n, \tcor, \thresh)$ on the public parameters $\pp$.
  $\A$ chooses the initial set of corrupted parties $\corrupt \gets \set{n}, \lvert \corrupt \rvert \leq \tcor$.
  $\SIM$ sets $\honest \gets [n]~\backslash~\corrupt$ and must reveal the secret keys of the corrupted parties to $\A$,
  which it does in the next step.

  \medskip

  \noindent\textit{Simulating KeyGen.}
    To simulate key generation where $\SIM$ is the trusted dealer,
    $\SIM$ does the following:
    \begin{enumerate}
    \item It sets $\pkgroup = X_0$.
    \item To simulate key generation for the signing keys,
    $\SIM$ first picks $t-1$ secret key shares for the corrupt parties
        $(\sk_i)_{i \in \corrupt} \rgets \Zq$,
    and defines their corresponding public keys
        $\pk_i \gets g^{\sk_i}$ for all $i \in \corrupt$.
    \item $\SIM$ then derives the public key shares
      $\pk_j, j \in \honest$ for the remaining honest parties in the exponent, as follows:
  \begin{align} \label{eq:keygen-sim}
    \pk_j = (\pk)^{L_0(j)} \cdot \prod_{i \in \corrupt} g^{\sk_i L_i(j)}
	\end{align}
        where $L_i(x)$ is the $i\supth$ Lagrange polynomial defined by $\corrupt \cup \{0\}$.
    \end{enumerate}


  Then,
  $\SIM$ runs
  $\adv^{ \BigO^{\Sign_1}, \BigO^{\Sign_2} }(\pkgroup, \{ \pkparty_j \}_{j \in \set{n}}, \{ \skparty_i \}_{i \in \corrupt},\state_A)$,
  and responds to its oracle queries,
  as we describe next.

  \medskip
  \paragraph{Simulating Random Oracles.}
  To simulate $\adv$'s random oracle queries,
  $\SIM$ programs $\HashNon, \HashSig$ with values from its input
  $\{h_1, \ldots, h_{\rototal} \}$,
  as follows.

  \begin{itemize}
  \item{}\underline{$\HashNon:$} When $\A$ queries $\HashNon$ on $(k, \threshpk, m, \{   (i, D_i, E_i)\}_{i \in \coalition})$,
  To respond,
  $\SIM$ does the following:
  \begin{enumerate}
    \item First, $\SIM$ checks if $\QNon[(k, \threshpk, \allowbreak m,\allowbreak \{ (i, D_i, E_i)\}_{i \in \coalition})] \neq  \bot$.
      If so, $\SIM$ simply returns $\QNon[(k, \threshpk, \allowbreak m,\allowbreak \{ (i, D_i, E_i)\}_{i \in \coalition})] $,
    \item Otherwise, for all $j \in \coalition$, $\SIM$ does the following:
  \begin{enumerate}
    \item $\SIM$ sets $a_j \gets h_\ell $ and
      $\QNon[(j, \threshpk, m, \{(i, D_i, E_i)\}_{i \in \coalition})] = a_j$
      where $h_\ell, \ell \in \set{\rototal}$ is the next-unused hash value provided to $\SIM$ on initialization.
  \end{enumerate}
    \item Then, $\SIM$ derives
    $\aggR \gets \prod_{i \in \coalition} D_i \cdot E_i^{a_i}$.
    \item $\SIM$ checks if $\QSig[(\aggR, \threshpk, m)] = \bot$;
      if the check does not hold,
      $\SIM$ aborts with $\BadEventOne$.
    \item Otherwise, $\SIM$ sets $c \gets h_{\ell+1}$,
      where $h_\ell$ is the next-unused random oracle output,
      and stores $\QSig[(\aggR, \pk, m)] = c$.
    \item Finally, $\SIM$ returns $a_k$.
  \end{enumerate}


  \item{}\underline{$\HashSig:$} When $\A$ queries $\HashSig$ on $(\aggR, \threshpk, m)$,
  $\SIM$ checks whether $\QSig[(\aggR, \threshpk, m)] \neq \bot$;
  if the check does not hold,
  then
  $\SIM$ returns $\QSig[(\aggR, \threshpk, m)]$.
	Else, $\SIM$ sets $c \gets h_\ell$,
  where $h_\ell$ is the next-unused random oracle output,
  stores $\QSig[(\aggR, \pk, m)] = c$ and returns $c$.
  \end{itemize}


  \medskip


	\smallskip

  \paragraph{Simulating Signing Oracles.}
  To simulate signing oracles,
  $\SIM$ does as follows:

  \begin{itemize}
    \item{Round 1 ($\BigO^{\Sign_1}(k, \ssid)$):}
	When $\A$ queries $\BigO^{\Sign_1}$ on honest participant identifier $k \in \honest$ and session identifier $\ssid$,
  $\SIM$ does the following:
  \begin{enumerate}
  \item Checks if this signing session has already started by checking if $(k, \ssid) \in W$,
  returning $\bot$ to $\adv$ if the check holds.
  \item Otherwise, it opens a new signing session for participant $k$ by storing $W \gets W \cup \{(k, \ssid) \} $.
  \item Next, $\SIM$ sets $D_k \gets X_\counter$ and $E_k \gets X_{\counter+1}$,
  where $\counter$ represents the next-unused value from the set of challenges $(X_1, \ldots, X_{2 q_S})$.
  It then increments $\counter \gets \counter + 2$.
  \item $\SIM$ then sets $\state_{k,\ssid} \gets (k, \ssid, D_k, E_k)$,
  and stores $Q_1[(k, \ssid)] = (D_k, E_k)$,
  \item Finally, $\SIM$ returns $(D_k, E_k)$ to $\A$.
  \end{enumerate}

\item{Round 2 ($\BigO^{\Sign_2}(k, \ssid, m, \coalition, \{ (D_i, E_i) \}_{i \in \coalition})$):}
	When $\A$ queries $\BigO^{\Sign_2}$ for honest participant identifier $k \in \honest$,
  session identifier $\ssid$, message $m$, signing set $\coalition$,
  and set of nonce commitments $\{ (D_i, E_i) \}_{i \in \coalition}$,
  $\SIM$ does as follows:
  \begin{enumerate}
  \item Checks if this signing session has already closed by checking if $(k, \ssid) \in W'$.
    If the check holds,
    return $\bot$ to $\adv$.

  \item Checks if this signing session has not yet been opened, by if $(k, \ssid) \not\in W$,
  returning $\bot$ if the check holds.
\item Checks that $(k, \ssid, D_k, E_k) \in Q_1$ and $Q_2[(D_k, E_k)] = \bot$
    to ensure that they were indeed issued in Round 1 and have not yet been used,
    returning $\bot$ if the check does not hold.
  \item Otherwise, $\SIM$ follows the protocol honestly to derive each $a_i, i \in \coalition$,
  and then the group commitment
  $\aggR \gets \prod_{i \in \coalition} D_i \cdot {E_i}^{a_i}$
  and the challenge $c \gets \HashSig(\aggR, \threshpk, m)$.
  \item $\SIM$ then sets $Z_k \gets D_k \cdot E_k^{a_k} \cdot {\pk_k}^{c \lambda_k}$,
  and queries the \AOMDL oracle $\odlsol$ on $Z_k$ and
  coefficients $(\alpha, ( \beta_i)_{i \in \set{2\qsig + 1}})$,
  where each coefficient is zero,
  with these exceptions;
  $\alpha = c \lambda_k \cdot \sum_{i \in \corrupt} \sk_i L_i'(k)$,
  and $(\beta_0 = c \lambda_k \cdot L_0'(k), \beta_\gamma = 1, \beta_{\gamma+1} = a_i)$.
  Here, $(\gamma, \gamma+1)$ corresponds to the \AOMDL challenges $(D_k=X_\gamma, E_k=X_{\gamma+1})$ for this signing query,
  where $L_i'$ is the Lagrange polynomial for $i \in \corrupt$ associated with the set $\{ 0\} \cup \corrupt$,
      such that $\pk_k^c = \pk^{L_0'(k) c} \cdot g^{c \alpha}$.
  \item $\SIM$ then receives $z_k$ in return from its \AOMDL oracle,
    which is the discrete logarithm of $Z_k$.
  \end{enumerate}
  \item $\SIM$ then stores $Q_2[(D_k, E_k)] = (z_k, c \lambda_k)$.
  \item $\SIM$ then returns $z_k$ to $\A$.
  \end{itemize}

  \medskip

  \paragraph{Analysis of Simulation.}
	$\SIM$'s simulation of key generation and signing is perfect.
  For key generation,
  $\A$ receives a valid secret key that is the point $i$ on the polynomial $f$,
  where $\pk_i = g^{f(i)}$.
  For signing, simulation is perfect because $\SIM$ receives $z_k$ from its \AOMDL oracle,
  which is the discrete logarithm of $Z_k$,
  with respect to $D_k, E_k, \pk_k$.


  $\SIM$'s simulation of the random oracles is perfect,
  with exception of when it aborts due to a bad event.
  $\SIM$ responds to random oracle queries by lazy sampling;
  $\SIM$ aborts due to $\BadEventOne$ with probability $1/q$ for each query to $\HashNon$,
  for a total of $\rototal$ possible queries,
  hence;
  it aborts with total probability $\rototal/q$ each time $\SIM$ is exeucted.


\paragraph{Output.} At the end of the game,
$\adv$ outputs a forgery $(m^*, \sigma^* = (R^*, z^*))$.
If $g^{z^*} \neq R^* \cdot (\pkgroup)^{c^*}$ (meaning that the forgery is invalid),
$\SIM$ outputs $\bot$.
Without loss of generality,
we assume that $\adv$ queried $\HashSig$ on $(R^*, \pkgroup, m^*)$.


Otherwise,
if $\adv$'s forgery is valid,
$\SIM$ outputs the tuple $(\ell, \aux)$,
where $\ell$ is the index corresponding to $c^* = h_\ell$,
and $\aux=(\{ \sk_i\}_{i \in \corrupt}, Q_1,  (m^*, \sigma^*))$ is the tuple
including the corrupted parties' secret key shares,
the set of signature shares in $Q_1$ output by $\SIM$ on behalf of honest parties during its simulation,
and the adversary's forgery with respect to $\pkgroup=X_0$.

  \paragraph{Difference between Game~1 and Game~2.}
  The only additional advantage introduced in Game~2 is that $\SIM$ aborts on   $\BadEventOne$
  with probability $\rototal/q$ total probability.
  Let $W_2$ be the event that $\adv$ wins in Game~2.
  Then

  \begin{equation} \label{eq:game2-frost}
    \lvert \Pr[W_2] - \Pr[W_1] \rvert \leq  \frac{\rototal}{q}
  \end{equation}


\medskip

  \paragraph{Finishing the Proof.}
  The advantage of $\adv$ against $\SIM$ is given by Game~2.
  Combining Equations~\ref{eq:game0-frost}, \ref{eq:game1-frost},
  and \ref{eq:game2-frost}  gives Equation~\ref{eq:frost-aomdl}.
  This completes the proof.
\end{proof}



\begin{proof}[Proof of Lemma~\ref{lem:frost-reduction}]
  We now show that if there exists a PPT adversary $\adv$ against the
  unforgeability of FROST and an algorithm $\SIM$
  as described in Lemma~\ref{lem:frost-simulation} that simulates FROST to $\adv$,
  then we can define a PPT adversary $\ddv$ that can efficiently solve for an \AOMDL challenge instance.

  $\ddv$ begins by accepting parameters $\pp = (\Gr, q, g)$ and a set  of $2\qsig+1$ \AOMDL challenges
  $\{ X_0, \ldots, X_{2\qsig}\}$.
  It then chooses a $(t, n)$ such that $n \geq t$.

  $\ddv$ then executes the modified forking algorithm $\modgenfork^{\SIM}(X)$ as described in Section~\ref{mod-fork} and shown in Figure~\ref{fg:gen-fork-mod},
  providing as input the instance $X = (\pp, \{ X_0, \ldots, X_{2\qsig}\}, n, t)$.
  Furthermore, $\ddv$ manages $\SIM$'s queries to the \AOMDL oracle.
  Each instance of $\SIM$ will ``use'' up to $\qsig$ queries to $\ddv$'s \AOMDL oracle,
  and so $\modgenfork^{\SIM}(X)$ will ``use'' $2\qsig$ \AOMDL oracle queries in total.

  $\modgenfork^{\SIM}(X)$ then either outputs $\bot$ when it fails or the value $(j, \aux)$ when it accepts.
  Corollary~\ref{cor:modified-fork} upper-bounds the probability that $\modgenfork^{\SIM}(X)$ will output $\bot$
  and Lemma~\ref{lem:frost-simulation} upper-bounds the additional advantage of $\adv$ against $\SIM$.

   We next show that when $\modgenfork^{\SIM}(X)$ outputs an accepting output
   (i.e., it does not output $\bot$),
   then $\ddv$ can solve for its \AOMDL challenge with perfect success.

 \paragraph{Extracting the Discrete Logarithm of $X_0$.}
   In the case that $\modgenfork^{\SIM}(X)$ outputs an accepting output,
   it outputs the tuple $(h_j, h_j', \auxstring, \auxstring')$.
   From these values,
   $\ddv$ then parses the following:

   \begin{align*}
     \begin{split}
       (m^*, \sigma^*)  \gets \auxstring,
       & \quad (m^{**}, \sigma^{**})  \gets \auxstring', \\
       (R^*, z^*)  \gets \sigma^*,
       & \quad (R^{**}, z^{**})  \gets \sigma^{**}, \\
       h_j \gets c^*, & \quad h_j' \gets c^{**}
     \end{split}
   \end{align*}

  Because $h_j, h_j'$ are with respect to the same index $j$ corresponding to
  $\adv$'s random query for $c^*$, $c^{**}$ in its first and second execution,
  then we know that $R^{*}=R^{**}$ and $m^{*}=m^{**}$.
   $\ddv$ can then solve for the discrete logarithm $y$ to its challenge $X_0 = g^y$,
   as follows:

   \begin{equation}
     x_0 = \skgroup =  \frac{z^* - z^{**}}{c^* - c^{**}}
   \end{equation}

   Because $\modgenfork$ outputs $\bot$ in the case when any random oracle outputs collide (i.e., in the case when any $h_i = h_i'$),
   then we know that this equation is solveable.

 \paragraph{Solving for Honest Parties' Secret Keys $\SK_k, k \in \honest$.}
 In order to perform the next step of solving for the remaining \AOMDL challenges
  $X_1, \ldots, X_{2\qsig}$.
 $\ddv$ needs to be able to derive the secret keys of each honest party.
  However, it can do so by using $\{ \sk_i \}_{i \in \corrupt}$ and $x_0$,
  simply by deriving
  $\sk_k \gets x_0 \lambda_0' + \sum_{i \in \corrupt} \sk_i \lambda_i'$,
  for each $k \in \honest$,
  where $\lambda_i'$ is the Lagrange coefficient associated with the set $\{ 0\} \cup \corrupt$.


 \paragraph{Extracting the Discrete Logarithms of $X_1, \ldots, X_{2\qsig}$.}
  Finally, $\ddv$ must solve for all remaining $(D_i=X_i, E_i=X_{i+1})$ \AOMDL challenges.
  $\ddv$ can do so in a same manner as prior reductions to \OMDL~\cite{BellareCKMTZ22},
  since $\ddv$ has already made its \AOMDL queries and obtained valid solutions.
  We give these techniques below, for completeness.


  How $\ddv$ solves for the discrete logarithm of each challenge pair depends on several factors.
  We describe each case.
  First,
  recall after the two (successful) executions of $\modgenfork^{\SIM}(X)$,
  $\ddv$ will obtain $(\auxstring, \auxstring')$,
  the signature pairs generated by $\SIM$ during its simulation of signing,
  as shown in Equation~\ref{eq:two-executions}.

  \begin{align} \label{eq:two-executions}
    \begin{split}
      & (\ssid_i, a_{i0}, c_{i0}, D_{i0}, E_{i0},  \resp_{i0}), i \in  \set{\numsignq},  \text{and} \\
      & (\ssid_i, a_{i1}, c_{i1},  D_{i1}, E_{i1}, \resp_{i1}), i \in \set{\numsignq} \\
    \end{split}
\end{align}


Because (1) $\SIM$ is deterministic
(as $\SIM$ is run on the same random tape for each execution, apart from the
values  $\{h_1, \ldots, h_{\rototal} \}$ to program its random oracles)
and (2) each commitment pair is associated with exactly one session identifier $\ssid_i$,
the commitment pairs $(D_{ij}, E_{ij})$  will be issued with the same $\ssid_i$ each time $\SIM$ is executed.
%
Hence, $D_{i0} = D_{i1}$ and $E_{i0} = E_{i1}$.

Consequently,
the following cases depend only on  the relation between
$(a_{i0}, a_{i1})$ and $(c_{i0}, c_{i1})$,
which in turn determine $(z_{i0}, z_{i1})$.

\begin{case}[$
a_{i0} \neq a_{i1},
c_{i0} \neq c_{i1}
$]
Because
  $a_{i0} \neq a_{i1}$,
  and $c_{i0} \neq c_{i1}$,
then
$\resp_{i0} \neq \resp_{i1}$ with overwhelming probability
(note that if
$a_{i0} \neq a_{i1}$ and $c_{i0} \neq c_{i1}$
but $\resp_{i0} = \resp_{i1}$,
$\odlsol$ will have been queried only once and so $\ddv$ can query the oracle directly
to solve,
similarly to what is described in the case below).

Hence,  in this case, $\odlsol$ will have been queried twice during the two executions of $\SIM$ to obtain $\resp_{i0} $ and $\resp_{i1}$
However,
$\ddv$ can solve for both  $x_i=d_i$ and $x_{i+1} = e_i$ without making any additional $\odlsol$ queries, by performing the following steps:

\begin{itemize}
  \item \textbf{Step One.} Derive $\resp_{i0} -\resp_{i1} $ (where unknown terms are underlined),
    resulting in Equation~\ref{eq:s_i-solve-1}, after cancelling out the $d_i$ term:

       \begin{equation} \label{eq:s_i-solve-1}
       \resp_{i0} - \resp_{i1} = \underline{e_i}\cdot a_{i0} + \SK_i \lambda_i \cdot c_{i0} - \underline{e_i}\cdot a_{i1} - \SK_i \lambda_i \cdot c_{i1}
   \end{equation}

    \item \textbf{Step Two.} $\ddv$ can then solve efficiently for $e_i$, by Equation~\ref{eq:solve-s_i-2}

    \begin{equation} \label{eq:solve-s_i-2}
        \underline{e_i} = \frac{ \resp_{i0} - \resp_{i1} + \SK_i \lambda_i ( c_{i1} - c_{i0})}{a_{i0}  - a_{i1}}
    \end{equation}

    Because $a_{i0} \neq a_{i1}$,
    Equation~\ref{eq:solve-s_i-2} is solveable.

    \item \textbf{Step Three.} $\ddv$ can then solve efficiently for $d_i$, as follows:

    \[ \underline{d_i} = z_{i0} - e_i \cdot a_{i0} - c_{i0} \cdot \SK_k  \]

\end{itemize}

So, in this case, exactly two $\odlsol$ queries were made for the two challenges $(X_i = D_i, X_{i+1} = E_i)$,
and $\ddv$ can efficiently solve for $y_i=d_i$ and $y_{i+1} = e_i$.
\end{case}

\begin{case}[$
a_{i0} = a_{i1},
c_{i0} = c_{i1}
$]

In this case,
because $a_{i0} = a_{i1}$ and
$c_{i0} = c_{i1}$,
then $\resp_{i0} = \resp_{i1}$.
Consequently,
$\SIM$ will have made only one unique $\odlsol$ to obtain $\resp_{i0} = \resp_{i1}$,
leaving one additional query available in the allowed ``budget.''


$\ddv$ can then query $\odlsol$ one additional time on $D_i$ to obtain $d_i=x_i$.
$\ddv$ can then solve for $x_{i+1} = e_i$ directly, by solving the linear relation in
Equation~\ref{eq:solve-s_i-3} where the single unknown term $e_i$ is underlined for emphasis:

\begin{equation} \label{eq:solve-s_i-3}
 \underline{e_i} = \frac{ \resp_{i0} - d_i - \SK_i \lambda_i \cdot c_{i0} }{a_{i0}}
\end{equation}

So, again, in this case,  exactly two $\odlsol$ queries were made for the two challenges  $(X_i = D_i, X_{i+1} = E_i)$, and $\ddv$ has solved for $y_i=d_i$ and $y_{i+1} = e_i$.
\end{case}

\begin{case}[$
a_{i0} \neq a_{i1},
c_{i0} = c_{i1}
$]

Because $a_i$ is (indirectly, via $\aggR$) an input to the random oracle when generating $c_i$,
then this event occurs only when there is a collision in $c_i$.
If this event occurs,
then $\ddv$ must abort.
We refer to this bad event as $\BadEventTwo$.
This case occurs with negligible probability $1/q$ for each signing query that $\adv$ makes,
with total probability $\numsignq/q$.
\end{case}


\begin{case}[$
a_{i0} = a_{i1},
c_{i0} \neq c_{i1}
$]
Again,
for similar reasons as in the case above,
when $\HashSig$ forks,
then subsequent queries to $\HashNon$ will also fork,
because $\SIM$ always programs its random oracle with the next-available value $h_i$.
We refer to this bad event as $\BadEventThree$.
%
This case occurs with negligible probability $1/q$ for each signing query that $\adv$ makes,
with total probability $\qsig/q$.
\end{case}


  \paragraph{Finishing the Proof.}

  $\ddv$ will hence succeed with advantage

    \begin{align*}
    \begin{split}
      \advant{aomdl}{\ddv^{\SIM}}(\secp)
      & \geq \Pr[ \modgenfork^{\SIM}(\pp, \{ X_0, \ldots, X_{2\qsig}\}, n, t) \neq \bot ]
        - \Pr[ \BadEventTwo ]
        - \Pr[ \BadEventThree ]
      \\
      & \geq
        \frac{\advant{uf}{\frost,\adv}(\lambda, n, t, t-1)^2}{\rototal}
        - \frac{2\rototal}{q}
        - \frac{3\rototal^2}{q}
        - \Pr[ \BadEventTwo ]
        - \Pr[ \BadEventThree ]
        \\
        & \qquad \qquad \qquad \qquad \text{\algcom  Combining Corollary~\ref{cor:modified-fork}
        and Lemma~\ref{lem:frost-simulation} }
        \\
        & \geq
        \frac{\advant{uf}{\frost,\adv}(\lambda, n, t, t-1)^2}{\rototal}
        - \frac{2\rototal}{q}
        - \frac{3\rototal^2}{q}
        - \frac{2 \qsig}{q}
    \end{split}
    \end{align*}

    First,
    Lemma~\ref{lem:frost-simulation} gives the relation
    $\Pr[ \SIM(\pp, n, t, t-1, \Inst) \neq \bot ]
    \geq
    \advant{uf}{\frost,\adv}(\lambda, n, t, t-1) -
        \frac{\rototal}{q}
        - \frac{\rototal^2}{2q} $.
%
    Then, because $\ddv$ runs the forking algorithm $\modgenfork$ on $\SIM$,
    and $\modgenfork$ runs $\SIM$ twice,
    then by Corollary~\ref{cor:modified-fork},
    $\Pr[ \modgenfork^{\SIM}(\pp, n, t, t-1, \Inst) \neq \bot ]$
    is greater than or equal to
    $
    \frac{\advant{uf}{\frost,\adv}(\lambda, n, t, t-1)^2}{\rototal}
    - \frac{2\rototal}{q}
    - \frac{2\rototal^2}{q}
    - \frac{2\rototal^2}{2q}
    $,
    or, simplified,
    $
    \frac{\advant{uf}{\frost,\adv}(\lambda, n, t, t-1)^2}{\rototal}
    -  \frac{2\rototal}{q}
    -  \frac{3\rototal^2}{q}
    $.
%
    This bound quantifies the probability that:
    \begin{enumerate}
      \item $\SIM$ will not abort across its two executions,
      \item $\SIM$ outputs the same index across its two executions, and
      \item none of the random oracle inputs collide.
    \end{enumerate}

    Then, we have to account for the probability that $\BadEventTwo$ and $\BadEventThree$ occur,
    which causes the reduction $\ddv$ to abort.
    Recall that
    $\Pr[\BadEvent_2] = \Pr[\BadEvent_2] = \qsig/q$,
    and each of these events occurs when $\ddv$ attempts to extract its \AOMDL solution
    (i.e., these events occur after $\modgenfork^\SIM$ has finished its execution).
    Hence, the probability of these bad events adds an additional $2\qsig/q$ term to yield the bound.

    This completes the proof.
\end{proof}
